# Vue3 视觉与交互落地方案

本文档用于指导 Vue3 重构完成后的界面统一优化。当前目标不是再次调整技术架构、接口契约或路由结构，而是在现有 Vue3 工程基础上统一视觉语言、交互模式和页面组件，使 `beacon-webmaster` 呈现为清爽、专业、高效的运营后台。

---

## 1. 设计目标

### 1.1 产品气质

`beacon-webmaster` 是短信平台运营控制台，界面应服务于高频配置、查询、审核、维护和问题排查。

目标气质：

1. 专业、克制、可信。
2. 信息密度高，但不压迫。
3. 表格清楚，操作路径短。
4. 表单稳定，错误提示明确。
5. 页面风格统一，不再出现 AdminLTE、Bootstrap、Layer、Element Plus 默认风格混杂。

避免方向：

1. 不做营销官网式大 hero。
2. 不做炫酷大屏风格。
3. 不使用大面积渐变、装饰图形、浮夸阴影。
4. 不把每个区块都做成厚重卡片。
5. 不在业务页面放说明性大段文案。

## 2. 视觉基础规范

### 2.1 色彩

建议在 `src/styles/tokens.scss` 中统一声明：

```scss
:root {
  --app-bg: #f5f7fb;
  --panel-bg: #ffffff;
  --panel-muted-bg: #f8fafc;

  --text-primary: #111827;
  --text-regular: #4b5563;
  --text-secondary: #6b7280;
  --text-placeholder: #9ca3af;

  --brand-primary: #2563eb;
  --brand-primary-hover: #1d4ed8;
  --brand-primary-light: #eff6ff;

  --success: #16a34a;
  --warning: #f59e0b;
  --danger: #dc2626;
  --info: #64748b;

  --border-light: #e5e7eb;
  --border-regular: #d1d5db;
  --shadow-soft: 0 1px 3px rgba(15, 23, 42, 0.08);
}
```

使用原则：

1. 主色只用于主按钮、当前菜单、关键链接和重点选中状态。
2. 成功、警告、危险、信息状态只用于状态表达，不用于装饰。
3. 内容区背景使用浅灰，页面主体面板使用白色。
4. 同一页面中不要出现多套蓝、绿、红。

### 2.2 字号

| 场景 | 字号 | 字重 |
| --- | --- | --- |
| 页面标题 | 20px | 600 |
| 区块标题 | 16px | 600 |
| 表格表头 | 13px | 600 |
| 正文/表格内容 | 14px | 400 |
| 辅助说明 | 12px | 400 |

原则：

1. 后台页面不使用超大标题。
2. 按钮、表格、表单字号保持稳定。
3. 不使用负字距，不根据视口宽度动态缩放字号。

### 2.3 间距与圆角

| 场景 | 建议值 |
| --- | --- |
| 页面外边距 | 20px / 24px |
| 区块间距 | 16px |
| 表单项间距 | 16px |
| 按钮间距 | 8px |
| 面板圆角 | 6px |
| 弹窗圆角 | 8px |
| 表格行高 | 44px / 48px |

原则：

1. 页面使用统一栅格和间距，不靠局部 `margin` 硬推。
2. 卡片圆角不超过 8px。
3. 不做卡片套卡片。

## 3. Layout 规范

### 3.1 整体布局

建议布局：

```text
AppLayout
 ├── Sidebar
 ├── HeaderBar
 └── AppMain
      └── RouterView
```

尺寸：

| 区域 | 建议 |
| --- | --- |
| Sidebar 展开宽度 | 220px |
| Sidebar 收起宽度 | 64px |
| Header 高度 | 56px |
| 内容区 padding | 20px / 24px |

### 3.2 侧边栏

1. 菜单层级最多视觉展示三层，超过三层需要调整信息架构。
2. 当前菜单项使用主色背景或浅主色背景，不使用强烈渐变。
3. 图标统一使用同一套图标库。
4. 菜单文字必须完整可读，收起态使用 tooltip。

### 3.3 顶部栏

顶部栏只放高频全局信息：

1. 面包屑。
2. 当前用户。
3. 退出登录。
4. 必要的全局快捷操作。

不要在顶部栏堆业务筛选项。

---

## 4. 页面结构规范

### 4.1 标准列表页

推荐结构：

```text
Page
 ├── PageHeader
 │    ├── title
 │    └── actions
 ├── ProSearch
 └── ProTable
```

规则：

1. 第一屏必须直接展示可操作内容。
2. 页面标题短而明确，如“客户管理”“通道管理”。
3. 主操作按钮放在 PageHeader 右侧。
4. 批量操作放在表格工具栏左侧。
5. 不在页面顶部放大段功能说明。

### 4.2 标准表单页

推荐结构：

```text
Page
 ├── PageHeader
 ├── FormSection 基础信息
 ├── FormSection 配置项
 └── FormActions
```

规则：

1. 表单按业务含义分区，不按控件类型分区。
2. 保存、取消按钮位置固定。
3. 长表单使用锚点或分区标题，避免一屏到底。
4. 必填、校验、错误提示全部使用 Element Plus 表单体系。

### 4.3 弹窗表单

弹窗宽度分三档：

| 类型 | 宽度 |
| --- | --- |
| 简单确认/简单编辑 | 480px / 560px |
| 常规编辑 | 720px |
| 复杂配置 | 960px |

规则：

1. 弹窗标题必须直接说明动作，如“新增客户”“编辑通道”。
2. 底部按钮统一右对齐。
3. 主按钮在右，取消按钮在左。
4. 删除、禁用、重置等危险操作必须二次确认。

---

## 5. 表格规范

后台系统的主要体验在表格，ProTable 必须统一。

### 5.1 表格视觉

1. 表头浅灰背景。
2. 行高统一为 44px 或 48px。
3. 鼠标悬停行使用极浅背景。
4. 数字列右对齐，文本列左对齐，状态列居中或左对齐但全系统统一。
5. 操作列固定在右侧。
6. 表格边框使用浅灰，不使用重边框。

### 5.2 操作列

1. 高频操作最多展示 3 个。
2. 次要操作进入“更多”下拉。
3. 危险操作使用红色文字或危险按钮，并二次确认。
4. 操作文案统一：新增、编辑、删除、启用、禁用、详情、重置。

### 5.3 状态展示

统一使用 `StatusTag`：

| 状态 | 类型 |
| --- | --- |
| 正常、成功、已启用 | success |
| 待处理、待审核 | warning |
| 异常、失败、已禁用 | danger |
| 未知、草稿、默认 | info |

禁止只用红绿文字表达状态。

### 5.4 空状态和加载态

1. 表格加载时使用 skeleton 或 table loading。
2. 空数据使用统一 EmptyState，文案为“暂无数据”。
3. 查询无结果时可提示“当前筛选条件下暂无数据”。
4. 接口异常时显示可重试按钮。

---

## 6. 搜索与筛选规范

### 6.1 ProSearch

1. 默认最多展示一行常用筛选项。
2. 超过 4 个筛选项时提供展开/收起。
3. 查询按钮为主按钮，重置按钮为普通按钮。
4. 日期范围、状态、客户、通道等高频条件位置保持一致。
5. 输入框回车触发查询。

### 6.2 筛选行为

1. 点击查询后分页回到第一页。
2. 点击重置后恢复默认查询条件并刷新表格。
3. 下拉选项加载失败时给出错误提示。
4. 不在用户输入过程中频繁自动请求，除非明确使用防抖。

---

## 7. 表单与校验规范

### 7.1 表单布局

1. 常规表单使用 label 左、控件右。
2. label 宽度统一，建议 96px / 120px。
3. 控件宽度统一，避免每一项长短不一。
4. 多列表单只用于字段很多且语义简单的场景。

### 7.2 校验反馈

1. 必填字段使用 Element Plus required 标记。
2. 错误提示显示在字段下方。
3. 提交失败后滚动到第一个错误字段。
4. 后端返回字段级错误时映射到对应表单项。

### 7.3 提交状态

1. 保存中按钮进入 loading。
2. 保存成功后给出成功提示，并关闭弹窗或返回列表。
3. 保存失败保留用户已输入内容。
4. 重复提交必须被按钮 loading 或请求锁拦截。

---

## 8. 反馈与异常规范

### 8.1 消息提示

| 场景 | 组件 |
| --- | --- |
| 成功保存、删除成功 | Message success |
| 普通失败、接口错误 | Message error |
| 危险操作确认 | MessageBox confirm |
| 复杂异常说明 | Result/Error 页面或弹窗 |

### 8.2 鉴权异常

1. 接口返回 HTTP `401` 时清理登录态。
2. 跳转登录页，并保留当前访问地址用于登录后回跳。
3. 禁止多个并发 `401` 同时弹出多条错误消息。

### 8.3 权限不足

1. 路由无权限进入 `/403`。
2. 按钮无权限时直接不展示。
3. 不使用 disabled 按钮暗示用户有权限但不可用。

---

## 9. 图表与数据展示规范

1. 图表页以读数和趋势为主，不做装饰性大屏。
2. 图表容器必须有稳定高度，避免 resize 时抖动。
3. 图例、tooltip、坐标轴格式统一。
4. 空数据时显示 EmptyState，不渲染空白图表。
5. ECharts 实例必须在组件卸载时销毁。

## 11. 落地节奏

由于 Vue3 重构已经完成，视觉优化按“样板先行、组件收口、页面批量统一”的方式推进。

### 11.1 样板页

先选 3 类页面作为视觉样板：

1. 登录页：确定品牌露出、表单、错误提示。
2. 典型列表页：如活动管理或客户管理，确定搜索、表格、分页、操作列。
3. 复杂表单页：如客户业务或通道配置，确定分区、校验、弹窗和提交状态。

样板页验收通过后，其他页面只允许复用样板模式，不再单独发挥。

### 11.2 批量统一

推荐顺序：

1. 全局 tokens 和 Element Plus 主题。
2. Layout、Header、Sidebar。
3. ProTable、ProSearch、StatusTag、ActionButtons。
4. 登录页。
5. 列表页批量统一。
6. 表单页和弹窗批量统一。
7. 图表页、空状态、异常页收尾。

---

## 12. 页面验收清单

每个页面完成视觉优化后，必须通过以下检查：

1. 页面第一屏可直接操作。
2. 页面标题、搜索区、表格区、操作区位置统一。
3. 表格行高、表头、分页、操作列一致。
4. 表单 label、控件宽度、错误提示一致。
5. 弹窗宽度、标题、底部按钮一致。
6. 状态字段使用统一 StatusTag。
7. 空状态、加载态、异常态完整。
8. `1366x768` 下不出现明显拥挤、遮挡和横向溢出。
9. 浏览器刷新、路由切换、弹窗打开关闭无明显抖动。
10. 不出现 AdminLTE、Bootstrap、Layer 旧风格残留。

---

## 13. 最终验收标准

当以下条件全部满足，可认为视觉与交互统一工作完成：

1. 所有页面使用统一 Layout 和主题变量。
2. 所有列表页使用统一 ProTable 和 ProSearch。
3. 所有表单页和弹窗遵循统一表单规范。
4. 所有状态、提示、确认、空状态和异常状态口径一致。
5. 关键页面在 `1366x768`、`1440x900`、`1920x1080` 下完成走查。
6. 产品、测试、前端共同确认样板页与批量页面风格一致。

视觉优化的最终目标不是“看起来热闹”，而是让运营人员长时间使用时更清楚、更稳定、更少误操作。
